## 笔记编写规范

1. **文件命名**：`XX-主题.md`，XX 为两位数序号，按知识板块排列（非按学习天数）
2. **内容结构**：问题驱动（场景 → 问题 → 原因 → 方案 → 代码 → 新问题 → 继续优化），末尾附总结表格
3. **风格一致**：使用 GitHub 风格表格、代码块标注语言类型、ASCII 架构图
4. **代码片段**：Java 为主（Spring Boot + Redis），标注关键注释
5. **新增笔记时**：同步更新 `README.md` 目录表格
