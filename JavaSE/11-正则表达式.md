# 11 - 正则表达式

## 1. 正则表达式概述

正则表达式（Regular Expression）是一种字符串匹配规则，用特殊符号定义查找模式。

**应用场景：**
- 校验用户输入（手机号、邮箱、身份证等）
- 文本搜索与替换
- 爬虫数据提取
- 日志分析

**Java 中的支持类：**
- `java.util.regex.Pattern` —— 编译正则
- `java.util.regex.Matcher` —— 匹配结果

---

## 2. 基本语法

### 2.1 字符类

| 表达式 | 含义 |
|--------|------|
| `[abc]` | 只能是 a、b 或 c |
| `[^abc]` | 除了 a、b、c 之外的任何字符 |
| `[a-z]` | a 到 z 的小写字母 |
| `[A-Z]` | A 到 Z 的大写字母 |
| `[a-zA-Z]` | 任意英文字母 |
| `[0-9]` | 任意数字 |
| `[a-zA-Z0-9]` | 任意字母或数字 |

### 2.2 预定义字符类

| 表达式 | 含义 |
|--------|------|
| `.` | 任意一个字符（换行符除外） |
| `\d` | 数字，等价于 `[0-9]` |
| `\D` | 非数字 |
| `\w` | 单词字符（字母、数字、下划线），等价于 `[a-zA-Z0-9_]` |
| `\W` | 非单词字符 |
| `\s` | 空白字符（空格、tab、换行等） |
| `\S` | 非空白字符 |

> ⚠️ Java 字符串中反斜杠需要转义，写 `\\d` 表示正则中的 `\d`

### 2.3 数量词（量词）

| 表达式 | 含义 |
|--------|------|
| `X?` | X 出现 0 次或 1 次 |
| `X*` | X 出现 0 次或多次 |
| `X+` | X 出现 1 次或多次 |
| `X{n}` | X 恰好出现 n 次 |
| `X{n,}` | X 至少出现 n 次 |
| `X{n,m}` | X 出现 n 到 m 次 |

### 2.4 边界匹配

| 表达式 | 含义 |
|--------|------|
| `^` | 行的开头 |
| `$` | 行的结尾 |
| `\b` | 单词边界 |
| `\B` | 非单词边界 |

### 2.5 逻辑运算符

| 表达式 | 含义 |
|--------|------|
| `XY` | X 后面紧跟 Y（拼接） |
| `X\|Y` | X 或 Y（或） |
| `(X)` | 分组，把 X 作为一个整体 |

## 3. Java 中使用正则

### 3.1 String 类对正则的支持

String 类本身就提供了几个正则相关的方法，日常开发最常用：

```java
// 1. matches() —— 判断是否完全匹配
String phone = "13812345678";
boolean ok = phone.matches("1[3-9]\\d{9}");  // true

// 2. replaceAll() —— 替换所有匹配
String msg = "你他妈的是个傻逼";
String clean = msg.replaceAll("他妈的|傻逼|操", "***");

// 3. replaceFirst() —— 替换第一个匹配
String str = "Hello World Hello";
str = str.replaceFirst("Hello", "Hi");  // "Hi World Hello"

// 4. split() —— 按正则切分
String csv = "张三,18,男";
String[] parts = csv.split(",");  // ["张三", "18", "男"]

// 去除多余空白
String messy = "a    b   c";
String[] items = messy.split("\\s+");  // ["a", "b", "c"]
```

### 3.2 Pattern 和 Matcher 类

需要更复杂的匹配操作时使用：

```java
// 1. 编译正则
Pattern pattern = Pattern.compile("(\\d{3})-(\\d{8})");

// 2. 创建匹配器
Matcher matcher = pattern.matcher("电话: 010-12345678");

// 3. 查找匹配
if (matcher.find()) {
    System.out.println("完整匹配: " + matcher.group(0));  // 010-12345678
    System.out.println("区号: " + matcher.group(1));      // 010
    System.out.println("号码: " + matcher.group(2));      // 12345678
}
```

**Matcher 常用方法：**

| 方法 | 说明 |
|------|------|
| `matches()` | 整个字符串是否完全匹配 |
| `find()` | 查找下一个匹配的子序列 |
| `group()` / `group(0)` | 返回匹配到的完整内容 |
| `group(n)` | 返回第 n 个分组捕获的内容 |
| `start()` | 匹配到的开始索引 |
| `end()` | 匹配到的结束索引（不包含） |
| `lookingAt()` | 从开头部分匹配（不需要完全匹配） |

### 3.3 爬取数据示例

```java
String html = "电话: 010-12345678, 手机: 13812345678, 邮箱: test@example.com";

// 提取所有手机号
Pattern p = Pattern.compile("1[3-9]\\d{9}");
Matcher m = p.matcher(html);

while (m.find()) {
    System.out.println("找到手机号: " + m.group());
}
```

## 4. 常见校验规则

```java
// 手机号 —— 13-19 开头，后跟 9 位数字
String regex = "1[3-9]\\d{9}";

// 邮箱
String regex = "\\w+@[\\w&&[^_]]+(\\.[a-zA-Z]{2,3})+";

// 身份证号（18位）
String regex = "[1-9]\\d{16}[\\dxX]";

// 固定电话（带区号）
String regex = "0\\d{2,3}-\\d{7,8}";

// 用户名（字母开头，4-16位字母数字下划线）
String regex = "[a-zA-Z]\\w{3,15}";

// 密码（6-16位，至少包含数字和字母）
String regex = "^(?=.*[a-zA-Z])(?=.*\\d).{6,16}$";

// IP 地址
String regex = "((25[0-5]|2[0-4]\\d|[01]?\\d\\d?)\\.){3}(25[0-5]|2[0-4]\\d|[01]?\\d\\d?)";

// URL
String regex = "https?://[\\w./?=&-]+";
```

> `(?=.*[a-zA-Z])` 是**零宽正向先行断言**——意思是从当前位置往后看，至少有一个字母，但它不消耗字符。

## 5. 贪婪模式与懒惰模式

```java
String text = "abbbc";

// 贪婪模式（默认）——尽可能多匹配
Pattern p1 = Pattern.compile("ab+");
Matcher m1 = p1.matcher(text);
m1.find();
System.out.println(m1.group());  // "abbb"

// 懒惰模式——尽可能少匹配
Pattern p2 = Pattern.compile("ab+?");
Matcher m2 = p2.matcher(text);
m2.find();
System.out.println(m2.group());  // "ab"
```

**量词的贪婪/懒惰版本：**

| 贪婪 | 懒惰 | 独占（Java 特有） |
|------|------|------|
| `X?` | `X??` | `X?+` |
| `X*` | `X*?` | `X*+` |
| `X+` | `X+?` | `X++` |
| `X{n}` | `X{n}?` | `X{n}+` |

**独占模式**：不会回溯，一旦匹配就钉死，不满足就立即失败。

## 6. 分组与反向引用

```java
// 捕获分组：用 () 包裹
String str = "hello hello world";
String result = str.replaceAll("(\\w+)\\s+\\1", "$1");  // "hello world"

// 反向引用 \1 表示重复第一个分组
String regex = "(.)\\1+";     // 匹配连续重复字符
System.out.println("aaabbb".matches("(.)\\1+"));  // false（完整匹配）
System.out.println("aaa".matches("(.)\\1+"));     // true
```

**分组类型：**

| 写法 | 类型 | 说明 |
|------|------|------|
| `(pattern)` | 捕获分组 | 捕获内容，可通过 group(n) 获取 |
| `(?:pattern)` | 非捕获分组 | 只分组不捕获，不占组号 |
| `(?=pattern)` | 正向先行断言 | 后面必须是 pattern |
| `(?!pattern)` | 负向先行断言 | 后面不能是 pattern |
| `(?<=pattern)` | 正向后发断言 | 前面必须是 pattern |
| `(?<!pattern)` | 负向后发断言 | 前面不能是 pattern |

**示例：**

```java
// 正向先行断言：提取后面是"元"的数字
Pattern p = Pattern.compile("\\d+(?=元)");
Matcher m = p.matcher("价格: 100元, 折扣: 50%");
m.find();
System.out.println(m.group());  // "100"

// 负向先行断言：提取后面不是"元"的数字
Pattern p2 = Pattern.compile("\\d+(?!元)");
// 匹配到 "50"
```

## 7. Pattern 的 flags

```java
// 不区分大小写
Pattern p1 = Pattern.compile("hello", Pattern.CASE_INSENSITIVE);

// 多行模式（^$ 匹配每一行的开头结尾）
Pattern p2 = Pattern.compile("^hello", Pattern.MULTILINE);

// 点号匹配换行符
Pattern p3 = Pattern.compile("h.+o", Pattern.DOTALL);

// 组合使用
Pattern p4 = Pattern.compile("hello", Pattern.CASE_INSENSITIVE | Pattern.DOTALL);
```

## 8. 性能建议

1. **复用 Pattern**：`Pattern.compile()` 耗资源，多次使用同一个正则应缓存
2. **少用 .\***：容易产生灾难性回溯
3. **用 `String` 的简单方法**优先于 `Pattern/Matcher`
4. **占位而非正则**：固定格式用 `String.format()` 或 `MessageFormat`，不要杀鸡用牛刀

```java
// 好的实践：缓存 Pattern
public class RegexUtils {
    private static final Pattern PHONE_PATTERN = Pattern.compile("1[3-9]\\d{9}");
    private static final Pattern EMAIL_PATTERN  = Pattern.compile("\\w+@[\\w&&[^_]]+(\\.[a-zA-Z]{2,3})+");

    public static boolean isValidPhone(String phone) {
        return PHONE_PATTERN.matcher(phone).matches();
    }
}
```

## 9. 总结

1. **正则的三种 Java 用法**：`String.matches()` / `String.replaceAll() / split()` / `Pattern & Matcher`
2. **三大核心概念**：字符类、数量词、分组
3. **贪婪 vs 懒惰**：默认贪婪，加 `?` 变懒惰
4. **性能关键**：复用 Pattern，避免灾难性回溯
5. **调试利器**：在线正则测试网站（regex101.com 等）验证后再写代码
