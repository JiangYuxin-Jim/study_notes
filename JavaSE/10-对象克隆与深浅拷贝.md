# 10 - 对象克隆与深浅拷贝

## 1. 对象克隆概述

对象克隆就是创建一个对象的副本，新对象和原对象拥有相同的属性值，但它们是不同的对象。

```java
// 以下不是克隆——只是引用的复制
Student s1 = new Student("张三", 20);
Student s2 = s1;  // s2 和 s1 指向同一个对象

// 真正的克隆需要创建一个全新的对象
```

Java 中实现克隆的两种方式：
1. **实现 `Cloneable` 接口 + 重写 `clone()` 方法**（浅拷贝）
2. **手动创建新对象并逐个赋值**（深拷贝）

---

## 2. Cloneable 接口与 clone() 方法

### 2.1 基本用法

```java
public class Student implements Cloneable {
    private String name;
    private int age;
    private Address address;  // 引用类型属性

    public Student(String name, int age, Address address) {
        this.name = name;
        this.age = age;
        this.address = address;
    }

    @Override
    protected Object clone() throws CloneNotSupportedException {
        return super.clone();  // 调用 Object 的 clone 方法
    }

    // getter / setter / toString ...
}

// 使用
Student s1 = new Student("张三", 20, new Address("北京"));
Student s2 = (Student) s1.clone();

System.out.println(s1 == s2);   // false —— 不同对象
System.out.println(s1.equals(s2)); // false —— 没重写 equals
```

### 2.2 注意事项

1. **必须实现 `Cloneable` 接口**——这是一个标记接口（没有方法），不实现会抛 `CloneNotSupportedException`
2. `clone()` 是 `Object` 的 `protected` 方法，需要重写为 `public`
3. 返回的是 `Object`，需要强转
4. `super.clone()` 执行的是**浅拷贝**

---

## 3. 浅拷贝 vs 深拷贝

### 3.1 浅拷贝（Shallow Copy）

`super.clone()` 默认就是浅拷贝：

- **基本类型**：复制值，没问题
- **String**：特殊，虽然也是引用但不可变，没问题
- **引用类型**：复制的是引用地址，新旧对象指向**同一个**引用对象

```java
Student s1 = new Student("张三", 20, new Address("北京"));
Student s2 = (Student) s1.clone();

s2.getAddress().setCity("上海");

// s1 的地址也变了！
System.out.println(s1.getAddress().getCity());  // 上海
System.out.println(s2.getAddress().getCity());  // 上海
```

### 3.2 深拷贝（Deep Copy）

所有引用类型的属性也要克隆，新旧对象完全独立。

**方式一：在 clone() 中手动深度克隆**

```java
@Override
protected Object clone() throws CloneNotSupportedException {
    Student cloned = (Student) super.clone();
    cloned.address = (Address) address.clone();  // Address 也要实现 Cloneable
    return cloned;
}

// Address 类也要实现 Cloneable
public class Address implements Cloneable {
    private String city;

    @Override
    protected Object clone() throws CloneNotSupportedException {
        return super.clone();
    }
}
```

**方式二：使用构造方法手动拷贝**

```java
public Student(Student other) {
    this.name = other.name;
    this.age = other.age;
    this.address = new Address(other.address.getCity());  // 新建对象
}
```

### 3.3 对比

| 特性 | 浅拷贝 | 深拷贝 |
|------|--------|--------|
| 基本类型 | ✅ 独立 | ✅ 独立 |
| String | ✅ 独立（不可变类） | ✅ 独立 |
| 引用类型 | ❌ 共享对象 | ✅ 独立对象 |
| 实现复杂度 | 简单（默认） | 较复杂 |
| 性能 | 快 | 较慢 |

## 4. 其他克隆方式

### 4.1 序列化实现深拷贝（不推荐，效率低）

```java
public Student deepClone() throws Exception {
    // 写入字节流
    ByteArrayOutputStream bos = new ByteArrayOutputStream();
    ObjectOutputStream oos = new ObjectOutputStream(bos);
    oos.writeObject(this);

    // 从字节流读取
    ByteArrayInputStream bis = new ByteArrayInputStream(bos.toByteArray());
    ObjectInputStream ois = new ObjectInputStream(bis);
    return (Student) ois.readObject();
}
```

要求：所有涉及的类都必须实现 `Serializable` 接口。

### 4.2 第三方工具（如 Apache Commons Lang）

```java
// 需要导入 Apache Commons Lang
Student s2 = SerializationUtils.clone(s1);  // 利用序列化
Student s3 = BeanUtils.cloneBean(s1);       // 利用反射
```

### 4.3 使用 Record（Java 14+）

Java 的 Record 默认是不可变的，适合用作数据传输，不需要考虑克隆问题。

---

## 5. 总结

1. **深浅拷贝的本质区别**：引用类型的属性是新对象还是和原对象共享
2. 开发中**浅拷贝够用就用浅拷贝**，不够说明类的设计有问题
3. 如果类全是基本类型和 String，浅拷贝就是深拷贝
4. **快捷键**：IDEA 可以自动生成 clone() 方法（Alt+Insert）
