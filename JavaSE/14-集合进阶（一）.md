## 一、集合 vs 数组

| 特性 | 数组 | 集合 |
|------|------|------|
| 长度 | 固定 | 可变 |
| 存储类型 | 基本类型 + 引用类型 | 只能存引用类型 |
| 功能 | 简单，操作有限 | 丰富（增删改查、排序等） |
| 效率 | 高 | 稍低 |

集合框架位于 `java.util` 包，是 Java 体系中最重要的部分之一。

## 二、集合体系结构

```
Collection（单列集合，存一个个元素）
├── List（有序、可重复、有索引）
│   ├── ArrayList（数组实现，查询快，增删慢）
│   ├── LinkedList（链表实现，增删快，查询慢）
│   └── Vector（古老，线程安全，已少用）
│
└── Set（无序（部分有序）、不可重复、无索引）
    ├── HashSet（哈希表，最快）
    │   └── LinkedHashSet（有序版）
    └── TreeSet（红黑树，可排序）

Map（双列集合，key-value 键值对）——后面再学
```

### 2.1 Collection 接口的共性方法

```java
Collection<String> coll = new ArrayList<>();

// 增
coll.add("Java");           // 添加元素，返回 boolean
coll.addAll(anotherColl);   // 添加另一个集合的所有元素

// 删
coll.remove("Java");        // 删除指定元素
coll.clear();               // 清空集合

// 查
coll.contains("Java");      // 是否包含
coll.isEmpty();             // 是否为空
coll.size();                // 元素个数

// 转数组
Object[] arr = coll.toArray();
```

## 三、List 集合

### 3.1 List 特有方法

List 因为有索引，所以多出了一套索引操作：

```java
List<String> list = new ArrayList<>();

list.add("Java");
list.add("Python");
list.add(1, "C++");        // 在索引1处插入（不会覆盖）

String s = list.get(0);    // 获取索引0的元素
list.set(0, "Go");         // 修改索引0的元素
list.remove(0);            // 删除索引0的元素（int 是索引）
list.remove("Java");       // 删除指定元素对象
list.indexOf("Python");    // 查找元素第一次出现的索引
list.lastIndexOf(x);       // 查找最后一次出现的索引
list.subList(from, to);    // 截取子列表 [from, to)
```

### 3.2 ArrayList 源码探秘（面试重点）

**底层结构：** Object[] 数组

**扩容机制：**
- JDK 7：创建时初始化长度 10 的数组，扩容为 1.5 倍 → 浪费内存
- JDK 8+：首次 `add` 时初始化长度为 10，扩容为 1.5 倍 → **懒加载**
- 扩容公式：`newCapacity = oldCapacity + (oldCapacity >> 1)`

```java
// JDK 8+ 核心扩容逻辑
private void grow(int minCapacity) {
    int oldCapacity = elementData.length;
    int newCapacity = oldCapacity + (oldCapacity >> 1);  // 1.5 倍
    if (newCapacity - minCapacity < 0)
        newCapacity = minCapacity;
    if (newCapacity - MAX_ARRAY_SIZE > 0)
        newCapacity = hugeCapacity(minCapacity);
    elementData = Arrays.copyOf(elementData, newCapacity);
}
```

### 3.3 LinkedList 源码探秘

**底层结构：** 双向链表

```java
// Node 内部类
private static class Node<E> {
    E item;           // 元素值
    Node<E> next;     // 后驱指针
    Node<E> prev;     // 前驱指针
}
```

**新增的双端队列方法：**
```java
LinkedList<String> list = new LinkedList<>();
list.addFirst("头");    // 在头部插入
list.addLast("尾");     // 在尾部插入
String first = list.getFirst();
String last = list.getLast();
list.removeFirst();
list.removeLast();
```

### 3.4 应用场景对比

| 场景 | 选谁 |
|------|------|
| 频繁按索引查询 | `ArrayList` |
| 频繁在头部/中间插入删除 | `LinkedList` |
| 只需要在尾部追加 | `ArrayList` |
| 实现队列/双端队列 | `LinkedList` |
| 只需要遍历元素 | 两者都可 |

## 四、泛型（Generic）

### 4.1 为什么要有泛型？

不用泛型时的问题：

```java
List list = new ArrayList();
list.add("Hello");
list.add(123);
list.add(new Student());

// 遍历时拿到的全是 Object，需要强制转型
String s = (String) list.get(0);  // 麻烦，且可能 ClassCastException
```

### 4.2 泛型的好处

```java
List<String> list = new ArrayList<>();
list.add("Hello");
// list.add(123);   // 编译错误，提前发现问题

String s = list.get(0);  // 不需要强制转型
```

**三大好处：**
1. 编译时检查类型安全
2. 消除强制转型
3. 提高代码复用性

### 4.3 泛型类

```java
// 定义泛型类
public class Box<T> {
    private T value;
    
    public void set(T value) {
        this.value = value;
    }
    
    public T get() {
        return value;
    }
}

// 使用
Box<String> stringBox = new Box<>();
stringBox.set("Hello");
String v = stringBox.get();
```

### 4.4 泛型方法

```java
public class Utils {
    // 泛型方法：<T> 声明在返回值前面
    public static <T> void printArray(T[] arr) {
        for (T item : arr) {
            System.out.println(item);
        }
    }
}

// 调用
Integer[] nums = {1, 2, 3};
String[] strs = {"A", "B"};
Utils.printArray(nums);  // 自动推断 T 为 Integer
Utils.printArray(strs);  // T 为 String
```

### 4.5 泛型接口

```java
// 定义
public interface MyList<E> {
    boolean add(E e);
    E get(int index);
}

// 实现方式1：知道具体类型
public class MyStringList implements MyList<String> {
    // 实现时 E 就是 String
}

// 实现方式2：延续泛型
public class MyArrayList<E> implements MyList<E> {
    // 调用者决定 E
}
```

### 4.6 泛型通配符

```java
// ? 表示不确定的类型
public void printList(List<?> list) {
    for (Object obj : list) {
        System.out.println(obj);
    }
}

// 上限通配符：? extends 类型
public double sum(List<? extends Number> list) {
    double total = 0;
    for (Number n : list) {
        total += n.doubleValue();
    }
    return total;
}
// 可传入 List<Integer>, List<Double>, List<Number>

// 下限通配符：? super 类型
public void addNumbers(List<? super Integer> list) {
    list.add(1);
    list.add(2);
}
// 可传入 List<Integer>, List<Number>, List<Object>
```

### 4.7 类型擦除

**泛型是编译期概念，运行时没有泛型**（JVM 不知道泛型的存在）。

```java
List<String> list1 = new ArrayList<>();
List<Integer> list2 = new ArrayList<>();

System.out.println(list1.getClass() == list2.getClass());  // true
// 运行时都是 ArrayList.class
```

- 编译器在编译后，泛型信息被擦除（type erasure）
- 无限制类型 → 替换为 Object
- 有上限类型 → 替换为上限类型

## 五、三种遍历方式

### 5.1 普通 for 循环（List 特有）

```java
List<String> list = Arrays.asList("A", "B", "C");
for (int i = 0; i < list.size(); i++) {
    System.out.println(list.get(i));
}
```

### 5.2 增强 for 循环（foreach）

```java
for (String s : list) {
    System.out.println(s);
}
```

- 所有 Collection 都能用
- 底层使用 Iterator
- 遍历过程中不能修改集合结构（不能 add/remove），会抛 `ConcurrentModificationException`

### 5.3 迭代器（Iterator）

```java
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    String s = it.next();
    System.out.println(s);
    // 迭代器自己的 remove 是安全的
    // it.remove();
}
```

**迭代器的基本方法：**
- `hasNext()`：是否有下一个元素
- `next()`：获取下一个元素，指针后移
- `remove()`：删除当前元素

## 六、小总结

| 知识点 | 要点 |
|--------|------|
| 数组 vs 集合 | 长度固定 vs 可变，基本 vs 引用 |
| Collection 体系 | List 有序有索引，Set 无序无索引 |
| ArrayList | 数组实现，查询快，扩容 1.5 倍（懒加载） |
| LinkedList | 双向链表，增删快，支持双端操作 |
| 泛型 | 编译期安全，消除转型，类型擦除 |
| 遍历 | fori / foreach / Iterator |
| 并发修改问题 | foreach 中不能 add/remove，用 Iterator 的 remove |
